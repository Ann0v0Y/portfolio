designer, programmer

Website developing with database

sVap is a website that facilitates item exchanges among residents of Greater Vancouver. Users can browse items, communicate with each other, and arrange exchanges.

Proposal

The website caters to visitors and members. Visitors can browse posts by category and keyword, view the latest items, and explore details. Members can log in, send messages, select categories, and post items.

Technique Used

HTML, PHP, CSS, and JavaScript (jQuery) were used for front-end development. AJAX was employed for dynamic interactions. PHP processed requests and managed data in a MySQL database.

Data Requirements Analysis & ER Diagram

The "member" section handles user registration with email, preferred username, password, contact numbers, and Vancouver city location. Data is stored as varchars.

In "post," posts are identified by post_id, linked to user email. They include title, description, timestamp, and category for filtering. Decimal item values and image_paths are added, while categories define trading intentions.

"Message" connects users via emails with timestamps and read status.

"Collection" records user-collected posts with email and post_id. This architecture creates a dynamic Vancouver-based trading platform, encouraging engagement and interaction.

Implementation details

Login

The website caters to visitors and members. Visitors can browse posts by category and keyword, view the latest items, and explore details. Members can log in, send messages, select categories, and post items.

HTML, PHP, CSS, and JavaScript (jQuery) were used for front-end development. AJAX was employed for dynamic interactions. PHP processed requests and managed data in a MySQL database.

Register

In the backend support, the backend checks if the provided email exists; if not, it hashes the password and adds data to the member table, responding with success.

For front end, errors display dynamically via JavaScript. On submit, AJAX sends data to the backend. If successful, the frontend redirects to the login page; otherwise, it displays messages using jQuery.

Browsing / Filter / Search

For backend support, the backend fetches and displays post data based on user search and filter input. Default shows all posts, while search/filter inputs refine results via dynamic queries. Rendered elements include a star button for non-post owners.

For front end, clicking a main category checkbox auto-selects subcategories. Search/filter button sends input to backend for AJAX response rendering posts via jQuery.

Add new post

For backend support, user input is used to insert data into post and want_to_trade tables. Images are stored, paths added to the database, and a success message is sent.

For front end, JavaScript handles dynamic image display, compression, and deletion. AJAX requests trigger page redirection or error message display based on backend response.

Chat

For backend support, the backend uses various commands like fetching contact information, messages, and unread counts, as well as inserting messages. Retrieved data is transformed into HTML elements and sent to the frontend. The "show user chat" command marks messages as read.

For front end, AJAX updates unread messages and user interactions, using jQuery to replace elements and render messages. User actions trigger AJAX requests to retrieve and display messages, and successful responses add chat messages to the container.

View Detail Page

For backend support, the backend retrieves detailed post information from the database based on a get request. It responds with HTML elements for frontend rendering, including a like/unlike button if the user is logged in but not the post owner, or a delete button for the owner.

For front end, clicking the like button sends an AJAX request to add the post to the database; if successful, the button changes. Clicking delete triggers a delete request and redirects to the main page. Clicking the message button directs the user to initiate a chat on the chat page.

Add/Remove Favorite Posts /View Favourites

For backend support, the backend handles add or remove favorite post requests by executing queries to insert/delete post IDs with user emails in the collection database. Response includes a new string with a list of collection post elements for operations via the favorite page.

For front end, clicking like/unlike buttons triggers AJAX requests to modify the database. Successful response updates the button appearance. On the favorite list page, AJAX response replaces and displays the new list of post items in the container.

Delete Post/View my post

For backend support, when an add or remove post request is received, the backend executes queries to insert/delete data associated with the input post ID from various tables. It also deletes related image files from the user's folder. The backend then fetches the user's updated post list and responds with HTML elements representing the posts.

For front end, clicking the delete button triggers an AJAX request to the backend for data deletion. Upon success, Jquery replaces list container elements with the updated post data, reflecting the changes on the webpage.
