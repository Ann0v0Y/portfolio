We created several functions for this accounting app by using Android Studio. Including basic functions for user to record their balance, keep their own records and other extra functions to help them use the app easier.

Basic Function

Sign in & Sign up

For users to register their accounts and used shared preference to store the user information so that they can view their previous receipts when they log in. The shared preference was also used for getting the data of specific receipts of a specific user by checking the user_email.

Balance

For users to manage their receipts by using a recycler view to show it. Users can add a new receipt by clicking the add button and they need to choose only one type of either expenditure or income, date and amount to save the receipt successfully. Otherwise, toasted messages will hint to them to complete the missing one.

Adding page

Users can only select one type for a record . Otherwise the selection will be cleared when they click save, accompanied by a toast message.

A popup window for the user to select a date, default as the current date. The user can only select today's date or the previous day's date.

The Amount is for users to enter numbers using the numeric keypad and provides comments for the user to enter whatever they want to record.

Once users are done and click the Done button, they can save the receipt and the receipt will be displayed in the previous page.

Standard Function

Calculator

Users can either use the calculator to calculate the amount by entering the page via homepage or adding page.

After getting the amount, user can click KEEP IT to save the number directly.

Users can click ‘back’ to back to homepage as well.

Map

Users can either use the map to record comment by entering the page via homepage or adding page. When they are around ambient environment, especially when they are traveling, which probably are the times when they spend the most and records are most necessary.

The initial location is the current location of the user.

Users can search the specific location by input address.

Finding page

The activity mainly relying on implicit intent. Our original intention is to allow users to keep track of their bills while also getting some real-time financial news and financial knowledge through our software. That way they can better manage their finances.

There’s also a link to live exchange rate queries in case users might have receipts of foreign exchange receipts and expenditures.

light&Night Mode

Users can log out in account page.

The mode will switch when light constant reach to 20000 lux.

Challenge

During the implementation of our bookkeeping application, we encountered an interesting challenge related to the display of receipt data from multiple users. Specifically, we found that all receipt data from different users were being shown together in the RecyclerView since we had just users table and receipts table in our database, which was not ideal.

Solution

Creating a separate table for each user is not a practical solution as it would lead to severe data redundancy and make managing and organizing data difficult. Instead, we implemented a foreign key SQLite statement that joins the receipts table to a separate user table. By adding user_email as a primary key to the user table and a foreign key to the receipts table, we were able to link each receipt record to the corresponding user record. This allows users to view receipt data for their account only.

This solved the challenge of displaying receipt data from multiple users and also improved the overall organization and management of data in our application.

Summary

In developing a bookkeeping application using Android Studio, we learned several important skills that are applicable to many different types of app development. For example, utilize the Google Maps API to track the user's current location, implement both implicit and explicit intents to navigate between different app activities and features, use shared preferences and SQLite database to store and retrieve user data as well as manage the database reasonably and implement a RecyclerView to present receipts and related data in an organized and user-friendly manner.

Overall, this project helped us develop our skills in Android development, programming, problem-solving, and testing and debugging for building a real-world application. These skills are highly valuable for us as aspiring app developers
